# Coop-Project

> Репозиторий для создания проекта по мобилке
Понял! Это важное уточнение. Теперь у нас получается иерархическая структура: **Мастера**, а внутри каждого мастера — его **Услуги**. Это логичнее, так как у разных мастеров цены на одну и ту же услугу могут отличаться (или набор услуг разный).

Вот обновленная структура данных и логика работы с учетом этого изменения:

---

### 1. Обновленная структура данных (Сущности)

Теперь главная единица каталога — это Мастер.

*   **Мастер (Master)**
    *   `Name` (Имя мастера).
    *   `ServicesList` (Список услуг, которые выполняет этот мастер).
*   **Услуга (Service)** — *теперь находится внутри Мастера*
    *   `Title` (Название: Маникюр, Стрижка).
    *   `Price` (Стоимость).
    *   `Description` (Описание).
*   **Пользователь (User)**
    *   `Login`, `Password`.
    *   `BookedServices` (Список записей: тут теперь важно хранить не только название услуги, но и к какому **Мастеру** записан клиент).

### 2. Работа с данными (JSON)

Файл `Catalog.json` (или `Masters.json`) теперь будет иметь вложенную структуру.
*Примерно так:*
```json
[
  {
    "Name": "Анна",
    "Services": [
       { "Title": "Маникюр", "Price": 1500 },
       { "Title": "Педикюр", "Price": 2000 }
    ]
  },
  {
    "Name": "Олег",
    "Services": [
       { "Title": "Стрижка", "Price": 1200 }
    ]
  }
]
```

### 3. Логика работы (Как меняются методы)

#### Каталог и Фильтрация
*   **DisplayAllData**: Теперь логичнее сначала выводить список Мастеров.
*   **Фильтрация по Мастеру**:
    1.  Пользователь выбирает Мастера из списка.
    2.  Программа показывает только список `Services` этого конкретного объекта `Master`.
*   **Поисковик**:
    *   Теперь поиск должен пробегаться по всем Мастерам, заглядывать в их списки услуг и искать совпадение по Названию услуги.
*   **Сортировка (по названию/цене)**:
    *   Если мы внутри меню конкретного Мастера — сортируем только его услуги.
    *   Если мы в "Общем поиске" — собираем услуги всех мастеров в один временный список и сортируем его.

#### Запись и Отмена
*   **Запись**: Когда пользователь бронирует услугу, мы сохраняем пару данных: `{MasterName: "Анна", Service: "Маникюр"}`.
*   **Отмена**: Удаляем эту связку из профиля пользователя.

### 4. Авторизация и Утилиты (Без изменений)
*   `resetFields`, `displayMessage`, `editSelectedUserPassword` — остаются такими же, так как это служебные функции, не зависящие от структуры каталога.
